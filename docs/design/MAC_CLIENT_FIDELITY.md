# Mac 控制端视觉验证记录

验证方式：Vite 本地服务 + Playwright（内置浏览器工具当前不可用，因此使用已安装的 Playwright 与本机 Chrome）在 1600×1000 视口截图；概念稿和实现截图均通过图像查看器逐项检查。

| 检查点 | 概念规格 | 实现结果 |
|---|---|---|
| 信息架构 | 左侧导航、项目头、阶段轨、指标、剧集、异常、详情 | 顺序和区域关系一致 |
| 配色 | 真白主面、石墨侧栏、朱红强调、绿色成功 | 一致，无额外渐变或玻璃效果 |
| 容器 | 开放布局、细分隔线、避免卡片堆叠 | 一致 |
| 字体与密度 | 中文系统无衬线、紧凑控制文字、醒目项目标题 | 一致；适配 1440px 以上生产窗口 |
| 媒体处理 | 16:9 剧情帧，无文字覆盖 | 使用项目生成的独立审核画面 |
| 交互 | 开始分析、异常切换、标记处理 | 浏览器验证通过 |

首屏文案差异：未添加概念外营销文案、眉题、徽章或伪指标；项目元数据根据工程 quality profile 使用 `research_best`。

有意偏差：应用定位为 macOS 桌面控制端，最小宽度为 1024px，不提供手机布局；宽度低于 1320px 时隐藏右侧审核区，保留剧集主流程。
