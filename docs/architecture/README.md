# 架构说明

VTV 分为五层：Mac 控制端、控制平面、编排与状态层、Modal 计算平面、数据与产物层。

1. Mac 控制端负责媒体探测、分片上传、项目配置、预览和人工审核。
2. 控制 API 负责认证授权、CRUD、预签名 URL、异步命令与查询。
3. PostgreSQL 保存业务状态、DAG、lease、outbox、成本、审核和审计记录。
4. Modal Worker 按分析、音频、渲染、QC、合成分池，业务逻辑与 Modal 装饰器隔离。
5. S3 兼容对象存储保存原片、代理、中间产物、候选与最终交付物。

核心不变量：HTTP 长任务只提交不等待；Worker 幂等；产物不可变；业务采用结果由数据库事务唯一决定。
